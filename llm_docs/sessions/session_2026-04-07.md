### handoff model simplification

- after reviewing the failed run, the cleaner design was obvious:
  - there should be one real handoff condition
  - fuel depletion should be only the fallback condition
- updated `fly_standard_ascent(...)` accordingly:
  - primary handoff trigger is now `periapsis >= -30_000`
  - backup handoff trigger remains `remaining <= handoff_fuel_threshold`
  - removed the apoapsis-ceiling handoff trigger from the decision rule
- rationale:
  - apoapsis alone proved to be a poor signal for when the lower stage had done enough useful orbital work
  - periapsis recovery is a more direct mission-progress signal for deciding when to give the upper stage the delicate insertion job
- compile check still passes:
  - `python3 -m py_compile scripts/run_saved_rocket.py`

### bounded handoff window design

- the periapsis-only handoff model also overshot badly in live flight:
  - handoff happened near `apoapsis ~174 km`
  - periapsis had improved to around `-30 km`
  - conclusion: periapsis alone is not enough either
- replaced single-threshold thinking with a bounded operating window for `core_stage_finish`
- current first-pass model:
  - preferred handoff window:
    - periapsis between `-50 km` and `+10 km`
    - apoapsis between `90 km` and `120 km`
  - backup handoff:
    - lower-stage fuel near empty
  - safety handoff:
    - apoapsis `>= 140 km`
    - periapsis `>= -100 km`
- throttle law inside `core_stage_finish` was also simplified to match the window idea:
  - `apoapsis < 90 km -> throttle 0.50`
  - `90 km <= apoapsis < 110 km -> throttle 0.20`
  - `apoapsis >= 110 km -> throttle 0.05`
