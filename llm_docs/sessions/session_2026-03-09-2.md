## design decisions — locked in before building

### fitness function
`delta_v` is the fitness score. rockets that fail structural validation or analytic filters score **0**. no partial credit.

- partial credit (scoring by how many checks pass) would help the GA bootstrap faster, but adds complexity and dilutes the signal once valid designs appear
- rejection sampling is fast enough that generation 0 should produce valid designs without scaffolding
- revisit only if generation 0 yields zero valid designs

```
fitness(rocket):
    if not validate_rocket(rocket):   → 0
    if not filter_rocket(rocket):     → 0
    else:                             → compute_delta_v(rocket)
```

### selection method
**tournament selection** (not fitness-proportionate / roulette wheel).

- FPS needs a wide continuous fitness distribution — early populations with many 0s and a few real scores cause FPS to degenerate or prematurely converge
- tournament is simpler, preserves diversity, and is robust to skewed score distributions

### build order
1. random rocket generator
2. fitness function stub (filters → delta-v or 0)
3. tournament selection
4. mutation operators
5. stage crossover
6. main GA loop
7. KSP harness integration (later)
