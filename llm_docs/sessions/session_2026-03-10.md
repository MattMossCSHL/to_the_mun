## design decisions locked in this session

### population structure
`population` is a list of `(rocket_dict, metadata_dict)` tuples.

`metadata_dict` always has `score` at minimum. `evaluate_population` takes a `detailed=False` flag:
- `detailed=False`: `meta = {'score': float}`
- `detailed=True`: `meta = {'score', 'valid', 'filtered', 'n_stages', 'n_parts', 'stage_dv', 'generation'}`

structure is always consistent — downstream code always does `rocket, meta = individual` and `meta['score']`. the flag only controls richness, not shape.

per-part data deferred — easy to add as another key later when there's a specific reason.

### per-generation export
- export format: JSON with generation number + list of rockets + metadata
- written at end of each generation in the main loop (not inside evaluate_population)
- output dir: `data/runs/run_YYYY-MM-DD-HHMMSS/gen_000.json`
- `save_generation(population, generation_num, path)` utility function planned

### progressive complexity
- idea A (increase max_stages on score threshold) — deferred, planned
- idea B (radial attachment) — deferred, later milestone
- see `llm_docs/design_decisions.md` for full writeup

## progress log

### evaluate_population — COMPLETE

signature: `evaluate_population(n_rockets, parts_by_name, resource_lookup, pods, tanks, engines, decouplers, max_stages=2, generation=0, detailed=False)`

- generates `n_rockets` random rockets, scores each one
- returns list of `(rocket_dict, meta)` tuples
- `meta` always has `score`; `detailed=True` adds `valid`, `filtered`, `n_stages`, `n_parts`, `stage_dv`, `generation`
- `stage_dv` computed via `compute_delta_v(return_breakdown=True)`, score derived via `sum(stage_dvs.values())` — avoids double call
- `stage_dv: {}` and `score: 0` when rocket fails validation or filters
- bugs caught: `np.sum` doesn't handle `dict_values` — use Python's built-in `sum`

### tournament_select — COMPLETE

signature: `tournament_select(population, pct_survivors=0.5, tournament_size=3)`

- draws `tournament_size` random competitors per round, keeps the highest scorer
- with replacement — same rocket can win multiple rounds
- `n_survivors = int(len(population) * pct_survivors)` computed internally
- zeros can survive if all competitors in a round score 0 — expected, they'll be bred out over generations
- extracted to `src/genetic_algorithm.py`

### mutate_swap_part — COMPLETE

signature: `mutate_swap_part(rocket_dict, pods, tanks, engines, decouplers)`

- deepcopies before modifying — original is never touched
- excludes pod_0 from candidates
- determines part category by membership in pods/tanks/engines/decouplers lists
- replaces with a random part of the same category
- bug caught: probe cores have resources + no engine so they appear in tanks list — pod exclusion by id is essential

### mutate_add_stage — COMPLETE

signature: `mutate_add_stage(rocket_dict, tanks, engines, decouplers, max_stages=4)`

- returns unchanged copy if already at max_stages
- finds bottom part via set difference of all_ids - parent_ids
- appends decoupler → tank → engine chain, each parented to the previous
- IDs use part-type counts to avoid collisions across multiple mutations
- updates stages dict for decoupler and engine
- bug pattern: `engine_` vs `eng_` prefix mismatch — existing parts use `eng_`

### mutate_remove_stage — COMPLETE

signature: `mutate_remove_stage(rocket_dict)`

- returns unchanged copy if single-stage (`max(stages.values()) == 0`)
- finds bottom engine via set difference, walks up two levels to get tank and decoupler
- removes all three from parts list and stages dict
- extracted to `src/genetic_algorithm.py`
