## analytic filters — IN PROGRESS

### context and motivation

the structural validator says "is this rocket possible." the analytic filters say "can this rocket plausibly fly." they sit between the validator and KSP in the pipeline and use cheap back-of-envelope physics to cull obviously bad designs. KSP is very slow and data will be scarce — these filters are critical.

### physics background

see `llm_docs/physics_primer.md` for full primer. key equations:

- **TWR** = F_thrust / (m_total × g) — must be > ~1.2 to lift off
- **delta-v** = Σ_stages [ Isp × g0 × ln(m_wet / m_dry) ] — must exceed mission budget
- **burn time** = m_propellant × Isp × g0 / F_thrust — sanity check per stage

### design decisions

- **resource densities scraped** from `GameData/Squad/Resources/ResourcesGeneric.cfg` via `parse_cfg` (same parser as parts scraper) — not hardcoded. user wanted to learn the scraping pattern.
- **parts list computed once** and passed into helpers — avoids recomputing `[part["type"] for part in rocket_dict["parts"]]` in every function
- **`resource_lookup` shape**: `{'LiquidFuel': {'density': 0.005}, ...}` — dict keyed by resource name, density in tonnes/unit
- **all top-level filter functions** take `(rocket_dict, parts_by_name, resource_lookup)` as signature

**what still needs to be built inside `compute_delta_v`**:

1. get Isp per engine: `parts_dict[part_type]['engine']['isp']['vacuum']`
2. compute m_wet per stage — start with total rocket wet mass, subtract jettisoned mass after each stage
3. jettisoned mass = mass of decoupler + all its descendants (parts below it in the tree)
   - decoupler for a given stage is found in `rocket_dict['stages']` — same stage number as the engine, type starts with `Decoupler`
   - descendant walk: start at decoupler, collect all children recursively (inverse of ancestor walk)
4. m_dry = m_wet - fuel_mass
5. delta_v for stage = isp × 9.80665 × ln(m_wet / m_dry)
6. accumulate total delta_v across all stages (high stage number → low)

**the question left hanging when we stopped**: "how do you identify which decoupler fires in a given stage?" — answer is in `staged_rocket['stages']`: the decoupler that shares a stage number with the engine is the one that fires. iterate over `stages.items()`, find parts in the same stage_num whose type starts with `Decoupler`.
