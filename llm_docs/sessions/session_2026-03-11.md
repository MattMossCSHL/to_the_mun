## progress log

### analyze_population тАФ COMPLETE (`src/analysis.py`)

signature: `analyze_population(population, top_n=10, verbose=False)`

- sorts population by score descending, slices top-N
- computes score stats (mean, median, max, pct_zeros), stage distribution, part count mean, and part-type Counters for both top-N and full population
- generates plain-English summary string
- returns `{'top': stats, 'full': stats, 'summary': str}`
- `verbose=True` prints the summary
- added as final cell in `notebooks/startup_GA_demo.ipynb`

### discovery: jet engine exploitation

running `analyze_population` on a GA result revealed:
- top 10 were all identical: `turboFanSize2` engine, `Size3SmallTank`, `cupola` pod, 1-stage, 3 parts, score 46,796 m/s
- `turboFanSize2` is an air-breathing jet engine тАФ very high Isp at low altitude but flames out ~25-30 km
- current delta-v formula uses stated Isp as if it applies the whole flight тАФ wildly inflates jet engine scores
- these rockets would take off and climb but never reach 70 km apoapsis

### decision: atmospheric delta-v model (not hard filter)

rather than blacklisting jet engines, model their actual contribution:
- detect air-breathing engines (propellant = `IntakeAir`)
- compute jet-stage delta-v contribution only up to operational ceiling (~25 km)
- hand off to rocket stages from that point
- jet engines remain valid parts (they're legitimately useful in lower stages of multi-stage designs)
- hard filtering was rejected: it throws away real design space and the oracle will naturally penalize designs that can't reach orbit

to be implemented as a new cell in `analytic_filters.ipynb`, then extracted to `src/filters.py`.
