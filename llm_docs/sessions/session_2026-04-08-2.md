  - `pods`
    - crewed only
    - stack-safe only
    - `vessel_type != 'Plane'`
    - must have both `top` and `bottom` nodes
  - `tanks`
    - no engine
    - not command parts
    - category limited to `FuelTank` / `Propulsion`
    - must contain supported tank propellants
    - must have both `top` and `bottom` nodes
    - must have matching top/bottom node sizes so adapters do not count as tanks
  - `engines`
    - must have engine data
    - category limited to `Engine` / `none`
    - propellants must be a subset of `{LiquidFuel, Oxidizer, SolidFuel}`
    - must have both `top` and `bottom` nodes

### runner staging diagnosis

- a live `run_saved_rocket.py` test on a top GA rocket suggested stage detachment / fuel monitoring might still be misaligned
- the test vehicle was a 3-stage stack with staged parts:
  - `eng_0 -> stage 0`
  - `decoupler_0 -> stage 1`
  - `eng_1 -> stage 1`
  - `decoupler_1 -> stage 2`
  - `eng_2 -> stage 2`
- the generated `.craft` staging looked internally consistent:
  - stage `0` for the top engine
  - stage `1` for the first decoupler
  - stage `2` for the middle engine
  - stage `3` for the second decoupler
  - stage `4` for the booster engine
- however, the runner log showed a mismatch between:
  - inferred designed stage propellants:
    - `{0: ('LiquidFuel',), 2: ('LiquidFuel', 'Oxidizer'), 4: ('SolidFuel',)}`
  - live KSP decouple-stage resource buckets:
    - propellant actually appeared in stages `3`, `1`, and `0`

### runner staging fix applied

- root cause:
  - `scripts/run_saved_rocket.py::project_stage_to_ksp_stage(...)` was mapping nonzero project stages to engine-action stages (`2 * stage`)
  - but `resources_in_decouple_stage(...)` groups fuel by the corresponding decouple stage, which is one lower for each nonzero stage
- fix applied:
  - changed the mapping to:
    - `0 -> 0`
    - `n > 0 -> 2 * n - 1`
- post-fix sanity check on the exact rocket produced:
  - `{0: ('LiquidFuel',), 1: ('LiquidFuel', 'Oxidizer'), 3: ('SolidFuel',)}`
- that now matches the live KSP stage-resource buckets observed in the verbose run
- compile check passed for `scripts/run_saved_rocket.py`
