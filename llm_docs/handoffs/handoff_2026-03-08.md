## analytic_filters.ipynb — IN PROGRESS

### what's done

resource densities scraped from `GameData/Squad/Resources/ResourcesGeneric.cfg` via `parse_cfg`. shape:
```python
resource_lookup = {'LiquidFuel': {'density': 0.005}, 'Oxidizer': {'density': 0.005}, ...}
```

TWR calculation complete via three helpers:
```
compute_twr(parts_list, parts_dict, fuel_lookup, g_const=9.80665)
    └── get_total_mass(parts_list, parts_dict, fuel_lookup)   # dry mass + fuel mass in tonnes
    └── calculate_thrust(parts_list, parts_dict)              # sum of max_thrust_kn
```
toy rocket TWR ≈ 3.93 (confirmed correct).

a two-stage test rocket `staged_rocket` is defined in the notebook for testing delta-v logic:
```python
# pod_0 → tank_upper → eng_upper → decoupler_0 → tank_booster → eng_booster
# stages: eng_booster=1, decoupler_0=1, eng_upper=0
```

parts library field names (confirmed):
- dry mass: `part['mass_t']` (float, tonnes)
- thrust: `part['engine']['max_thrust_kn']` (float, kN)
- isp: `part['engine']['isp']['vacuum']` and `['sea_level']` (float, seconds)
- resources: `part['resources']` — dict like `{'LiquidFuel': 180.0, 'Oxidizer': 220.0}` or `None`

### what's mid-build: `compute_delta_v`

the function is partially written. current working pieces:
- `id_to_type` and `id_to_parent` lookups from `rocket_dict['parts']`
- outer loop over stages, correctly skipping non-engines
- ancestor walk climbs tree from engine to root, stops at `Decoupler_*` type parts
- fuel mass accumulation working (2.0t per fuelTank confirmed correct)

**what still needs to be built**:

1. get Isp: `parts_dict[part_type]['engine']['isp']['vacuum']`
2. m_wet per stage — start with total wet mass, subtract jettisoned mass after each stage fires
3. identify jettisoned parts: the decoupler that shares a stage number with the engine + all its descendants (descendant walk = inverse of ancestor walk, collect children recursively)
4. m_dry = m_wet - fuel_mass
5. delta_v per stage = isp × 9.80665 × ln(m_wet / m_dry)
6. sum delta_v across all stages (highest stage number first, down to 0)

**the question left hanging**: "how do you identify which decoupler fires in a given stage?" — answer: iterate `stages.items()`, find parts in the same `stage_num` whose type starts with `'Decoupler'`. that's the one to jettison.

### after delta_v: remaining work

- `compute_burn_time(rocket_dict, parts_by_name, resource_lookup)` — fuel_mass × Isp × g0 / thrust per stage
- `filter_rocket(rocket_dict, parts_by_name, resource_lookup, goal)` — runs all three, returns `(bool, list_of_reasons)`
- test on toy rocket and bad designs
- extract to `src/filters.py`
