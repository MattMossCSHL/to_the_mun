## what was built this session

### functions in src/filters.py

```
get_total_mass(parts_list, parts_dict, fuel_lookup)
calculate_thrust(parts_list, parts_dict)
calculate_twr(parts_list, parts_dict, fuel_lookup, g_const=9.80665)
compute_delta_v(rocket_dict, parts_list, parts_dict, resource_lookup, g_const=9.80665, verbose=False)
compute_burn_time(rocket_dict, parts_list, parts_dict, resource_lookup, g_const=9.80665, verbose=False)
filter_rocket(rocket_dict, parts_list, parts_dict, resource_lookup, dv_thresholds, goal='orbit', g_const=9.80665, verbose=False)
DV_THRESHOLDS  # module-level dict: atmosphere/orbit/mun_orbit/mun_landing
```

### key design decisions

- `parts_list` is `[p['type'] for p in rocket_dict['parts']]` — caller computes it once and passes in
- `resource_lookup` shape: `{'LiquidFuel': {'density': 0.005}, ...}` — scraped from `ResourcesGeneric.cfg`
- `compute_delta_v` processes stages high → low (first to fire), tracks running `m_wet`, subtracts jettisoned dry mass after each stage
- jettisoned parts = decoupler (same stage_num, type starts with `Decoupler`) + all descendants (descendant walk via stack + `id_to_children`)
- `compute_burn_time` returns `{stage_num: seconds}` dict, not a scalar
- `filter_rocket` returns `(passed: bool, reasons: list[str])`
- `goal` flag drives delta-v threshold only; TWR floor (1.2) and burn time floor (5s) are fixed

two-stage test rocket: pod → tank_upper → eng_upper → decoupler_0 → tank_booster → eng_booster

## bugs caught this session (for next agent's awareness)

patterns that came up more than once:

- **`continue` condition inverted** — `if engine_id is not None: continue` skips stages with engines. easy mistake, check the logic direction.
- **loop variable reuse** — `part_id` from inner loop used outside it; always use the named variable (`engine_id`) not the loop variable
- **advance inside conditional** — `current = id_to_parent[current]` inside `if part_resources is not None` caused infinite loop on parts without resources. advance must be at while-loop level.
- **initialization inside loop** — `burn_times = {}` inside the stage loop resets every iteration. always initialize accumulators before the loop.
