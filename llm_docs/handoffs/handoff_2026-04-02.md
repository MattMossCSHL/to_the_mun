- launch engine (`eng_4`): `liquidEngine2-2_v2`

### key telemetry

The critical telemetry was:

- stage started at `8`

That means:

- the active launch stage was ignited

- this was not primarily a "pieces were unattached" problem

## real diagnosis

The selected GA winner is **not actually launch-capable**.

### actual launch-stage numbers

For the selected rank-1 rocket from `run_2026-04-02-221216`:

- launch engine: `liquidEngine2-2_v2`
- in this parts library that is the `Poodle`
- launch-stage sea-level thrust: `250 kN`
- total wet stack mass: about `63.41 t`
- actual launch TWR: about `0.40`

A rocket with TWR `0.40` cannot leave the pad, so the failure is expected even if the craft is assembled correctly.

## why GA still chose it

The current TWR filter in `src/filters.py` is too optimistic.

Right now:

- `calculate_twr(...)` sums the thrust of **all engines in the rocket**
- `filter_rocket(...)` uses that number as "launch TWR"

For the failed rocket:

- total thrust from every engine in the whole stack is about `1640 kN`
- that yields a misleading whole-rocket TWR around `2.64`
- but only the bottom / highest-numbered staged engine matters at liftoff in this linear-stack model

So the GA is currently allowed to score rockets that have excellent total theoretical thrust spread across future stages, while still being physically incapable of lifting off.

## first thing to do next session

Do **not** start with serializer work.

Do **not** start with Mun-transfer tuning.

Start by fixing the analytic launch-TWR filter in `src/filters.py`.

### recommended fix

In the current linear-stack rocket model:

- identify the highest-numbered staged engine in the rocket
- treat that as the launch engine
- compute launch TWR using:
  - that engine's thrust
  - divided by full wet stack weight

Then reject rockets whose true launch-stage TWR is below threshold.

That should stop GA from selecting "Poodle as first stage" rockets.

## after that

Once launch-stage TWR is fixed:

1. rerun GA
2. select a new rank-1 rocket
3. test `scripts/run_saved_rocket.py --mission mun_orbit ...`
4. only if that new rocket still fails, decide whether the next issue is:
   - serializer / attachment
   - ascent / orbit logic
   - transfer / capture logic
